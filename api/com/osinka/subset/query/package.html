package query

Field Queries

Typical query starts from a field. The simplest condition is an equality test (see FieldConditions).

It is possible to create several tests on the same field (an example is a range f > 3 <= 10) or negate a condition, seeFieldQuery for details.

Composing Queries

It is possible to compose queries with &&, || and nor methods. See Query for details.

Subset

com.osinka.subset.Subset.elemMatch method allows creating \$elemMatchqueries, when you need to match all the conditions on a specific subdocument.

val query = BlogPost.comments.elemMatch { comment =>
  comment.by === "joe" && comment.votes === 2
}

val blogPosts = collection.find(query)
