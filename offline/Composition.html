Subset — Composition

Composition

Query is built on top of Mutation and benefits from the ability of mutations to modify existing DBObject.

Note While queries can be composed together using ~ method, the result will be Mutation and cannot account for the possible fact this query is in scope of a Subset, thus will be lacking MongoDB “dot notation” to designate internal keys. It’s better to use special methods for composing queries, outlined below.

Query has the following methods to compose:

  • def ||(other: Query) (the same as or) creates $or queries: ”q1 || q2 || q3” results in ”{$or: [q1, q2, q3]}”
  • def &&(other: Query) (the same as and) creates “and” query. The specific behaviour depends on the strategy. By default, Subset tries to pack all fields in a single DBObject, but when any of the queries makes use of $and or same field names are used, $and query will be used.
  • def nor(other: Query) creates $nor quries.
